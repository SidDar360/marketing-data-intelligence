## Project Overview

Marketing Data Intelligence — an e-commerce ML system built on the Amazon Sales Dataset (`amazon.csv`, 1465 products). Implements: (1) discount/price prediction models, (2) an LLM-powered RAG assistant, (3) a FastAPI service containerized with Docker.

## Workflow

**Step 1 — Train everything:**
```bash
python train.py
```
Outputs model artifacts and FAISS index to `artifacts/`. Must run before starting the API.

**Step 2 — Start the API:**
```bash
uvicorn src.api:app --reload
```

**Or run with Docker:**
```bash
docker compose up --build
```

**Run tests:**
```bash
pytest tests/
```

**Run a single test file:**
```bash
pytest tests/test_api.py -v
```

## Architecture

```
train.py              Entry point: trains models + builds RAG index sequentially
src/
  data_preprocessing.py  load_and_clean_data(), get_feature_target_for_*(), save_training_stats()
  models.py              train_random_forest(), train_linear_regression(), check_drift(), save/load_model()
  rag.py                 build_document_corpus(), build_faiss_index(), retrieve_relevant_docs(), generate_answer()
  api.py                 FastAPI app — /predict_discount, /answer_question
tests/                 Unit tests for each module
artifacts/             Generated by train.py (gitignored) — .pkl models, faiss_index, training_stats.json
```

**Data cleaning** (`data_preprocessing.py`): Price fields arrive as `"₹1,299"` strings. `load_and_clean_data()` strips `₹`/commas → float; strips `%` and non-numerics from rating/discount → float; drops rows missing any key numeric field.

**Predictive models** (`models.py`):
- `RandomForestRegressor` → predicts `discount_percentage` from `[actual_price, discounted_price, rating, rating_count]`
- `LinearRegression` → predicts `discounted_price` from `[actual_price, rating, rating_count, discount_percentage]`
- `check_drift()` computes per-feature z-scores against `artifacts/training_stats.json` to flag out-of-distribution inputs

**RAG assistant** (`rag.py`): Builds text documents per product (name, category, prices, about, review) → encodes with `all-MiniLM-L6-v2` → stores in FAISS IndexFlatL2. At query time: retrieves top-k docs → constructs instruction prompt → generates with `google/flan-t5-base` (override via `LLM_MODEL` env var). Pipeline is cached in a module-level dict after first load.

**API** (`api.py`): All heavy state (models, FAISS index) is loaded once in the `lifespan` context manager into `app.state`. Route handlers access via `request.app.state`. Missing artifacts → 503 with actionable error message.

## API Endpoints

| Method | Path | Body | Returns |
|--------|------|------|---------|
| GET | `/health` | — | `{"status":"ok"}` |
| POST | `/predict_discount` | `{actual_price, discounted_price, rating, rating_count}` | `{predicted_discount_percentage, drift_warning, drift_details, inputs}` |
| POST | `/answer_question` | `{query, top_k=3}` | `{question, answer, sources}` |

## Environment Variables

| Var | Default | Purpose |
|-----|---------|---------|
| `LLM_MODEL` | `google/flan-t5-base` | Any HuggingFace text2text-generation model |
